Dashboard & Visibility

How to use Trak's dashboard to see everything at once

What's there by default, what you configure, and how the Everything Table and instant charts give you a complete picture of your workspace.

Start free Book a walkthrough
How it works
Trak has four visibility layers: the workspace dashboard (configurable, starts useful from day one), the Everything Table (its own page — every item in your workspace, filterable), instant charts (any data type, any format, anywhere), and per-project overviews (scoped to a single project). Each serves a different level of zoom.
Step 1

The workspace dashboard — useful from day one, no setup needed

1
Open the dashboard — everything default is already there
Click Dashboard in the sidebar. Four things are there automatically: an AI workspace overview, task cards (Due Today, Due Soon, Overdue), a notifications feed, and a project status list. Nothing to configure to get here — it pulls from everything in your workspace from the moment you start using Trak.
2
The AI overview reads across every project and summarizes it
At the top of the dashboard, the AI reads your tasks, deadlines, comments, and file activity across every project and writes a plain-language summary. Which projects are on track. What's overdue. Where there's been recent feedback. Hit Regenerate any time for a fresh read — it also refreshes automatically on a schedule.
3
Task cards and notifications are always live
The Due Today, Due Soon, and Overdue cards pull from every task block in every project — always current, no manual updates. The notifications feed shows team activity and client feedback as it comes in, labeled separately so you can tell internal from external at a glance. Click anything to go directly to its source.

Step 2

Configure the dashboard to show exactly what your team needs

1
Click + Add to start building on top of the defaults
Scroll below the default cards and click + Add. A picker opens with everything you can add — chart blocks, quick links to projects, custom sections. Add as many as you want. The dashboard is a canvas. You build on top of the defaults to make it useful for how your team actually works.
2
Add quick links to your most active projects
Select Quick Link → Project and choose any project. It appears as a card showing the project name, status, due date, and open task count — live and always current. Add one for every project your team is actively running and click any card to jump directly in.
3
Rearrange, remove, and make it yours
Drag any section to reorder it. Remove cards you don't need. If quick links are what your team uses most, move them to the top. If you only need the AI overview and one chart, keep it minimal. The layout is persistent — it stays exactly the way you configure it.

Step 3

The Everything Table — every item in your workspace, filterable by any property

1
Click Everything in the sidebar
The Everything Table is its own page in Trak — click Everything in the sidebar. It pulls every item across your entire workspace — every task, every block that has properties set — into a single table. Every row shows the five universal properties: status, priority, tags, due date, and assignee. Nothing is excluded.
2
Filter it down to exactly what you want to see
Click Filter and combine any properties — assignee is Mia, priority is Urgent, tag is packaging. Stack as many filters as you need. The table updates instantly. The result is a precise, live slice of your workspace — every matching item across every project, pulled into one view.
3
Click any row to go directly to that item
Every row in the table links back to its source — the block, the tab, the project. The Everything Table is a view into your workspace, not a copy of it. When you click through, you land in context with everything around it visible.
Most useful for: pre-launch audits ("everything tagged launch that isn't done"), workload checks ("everything assigned to Jade due this week"), and cross-project reviews ("all urgent items across the workspace").

Step 4

Instant charts — any data type, any format, anywhere in Trak

1
Add a chart block from the dashboard or any project tab
Charts aren't just a dashboard feature. Add a chart block to the dashboard, to a project's overview tab, or to any canvas tab inside a project — anywhere you'd want to see data visualized inline. Click + → Chart block wherever you are.
2
Choose what data to visualize
Charts can be scoped to any data type in Trak — tasks by status, tasks by priority, tasks by assignee, tasks by tag, completion rate over time, items by project, overdue count — or any combination. You're not limited to project-level rollups. A chart can pull from a single tab, a specific tag, a specific assignee, or everything at once.
3
Pick the format that communicates it best
The same data can be displayed multiple ways. Bar for comparing counts across categories. Donut for showing distribution at a glance. Line for tracking change over time. Summary count for a single number stat card. Choose the format for the insight, not just the aesthetic.
Charts are live. They pull from your workspace data in real time — as tasks are completed and properties change, the charts update automatically. No exports, no manual refreshes.

Step 5

Every project has its own overview — the same dashboard, zoomed into one project

1
Click Overview in any project's tab bar
Every project has an Overview tab that's always there — you don't create it. Click it and you get the project-level version of the dashboard: open task count, Due Today, Due Soon, and Overdue cards — all scoped to that project only. Same format as the workspace dashboard, zoomed in to a single project.
2
It also shows recent team comments from across the project
Below the task cards, the project overview surfaces a comment feed — who said what, on which block, with a direct link. Use it as your project-level async standup. Before a team check-in, open the overview. You're oriented in 20 seconds without opening individual tabs.
3
Each task in the overview links directly to where it lives
Click any task in the overview cards and you're taken to it in context — in its tab, in its block, with everything around it visible. The overview is a navigation layer, not a flat list. You're always one click away from the actual work.
